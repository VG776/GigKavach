## The 7 Handlers

### 1️⃣ `handle_join(phone, message)` - Start onboarding
```python
response = await handle_join("+919876543210", "JOIN")
# → "👋 Welcome! Reply with language: 1️⃣ English..."
```

**Triggers**: User sends "JOIN"
**Returns**: Language selection prompt
**Side Effect**: Initializes Redis state

---

### 2️⃣ `handle_status(phone, message)` - Check coverage + DCI
```python
response = await handle_status("+919876543210", "STATUS")
# → "📊 Zone 560001 Status: DCI: 45 (low), Coverage: Shield Plus ✅"
```

**Triggers**: User sends "STATUS"
**Returns**: Zone DCI + active plan
**Data**: Queries Supabase for worker + policy

---

### 3️⃣ `handle_renew(phone, message)` - Extend coverage
```python
response = await handle_renew("+919876543210", "RENEW")
# → "✅ Renewed! Coverage active from 2024-12-18"
```

**Triggers**: User sends "RENEW"
**Returns**: Renewal confirmation
**Side Effect**: Creates new policy week

---

### 4️⃣ `handle_shift_update(phone, message)` - Update work hours
```python
response = await handle_shift_update("+919876543210", "SHIFT 2")
# → "✅ Shift updated to Day!"
```

**Triggers**: User sends "SHIFT <1-4>"
**Returns**: Shift confirmation
**Side Effect**: Updates workers.shift

---

### 5️⃣ `handle_language_change(phone, message)` - Switch language
```python
response = await handle_language_change("+919876543210", "LANG 2")
# → "✅ ಭಾಷೆ ಕನ್ನಡಕ್ಕೆ ಬದಲಾಗಿದೆ!"
```

**Triggers**: User sends "LANG <1-5>"
**Returns**: Confirmation in new language
**Side Effect**: Updates workers.language

---

### 6️⃣ `handle_help(phone, message)` - Show commands
```python
response = await handle_help("+919876543210", "HELP")
# → "GigKavach Commands: • STATUS... • RENEW... • SHIFT..." 
```

**Triggers**: User sends "HELP"
**Returns**: Command menu (multilingual)

---

### 7️⃣ `handle_appeal(phone, message)` - Open dispute
```python
response = await handle_appeal("+919876543210", "APPEAL I was incorrectly flagged")
# → "✅ Appeal submitted. We'll review within 48 hours"
```

**Triggers**: User sends "APPEAL <reason>"
**Returns**: Appeal confirmation
**Side Effect**: Creates appeals record (48h expiry)

## The Main Router

```python
async def route_message(phone: str, body: str) -> str:
    """
    Dispatcher - routes incoming WhatsApp message to correct handler
    
    Logic:
    1. Parse keyword (first word, uppercase)
    2. Route to handler if keyword matches
    3. Else, check Redis for onboarding state
    4. Route to step-specific handler if onboarding
    5. Else, return HELP (default)
    """
```

**Example Call Flow**:
```
Twilio webhook receives message
  ↓
whatsapp.py extracts phone + body
  ↓
calls route_message(phone, body)
  ↓
route_message dispatches to correct handler
  ↓
handler returns response string
  ↓
whatsapp.py wraps in TwiML XML
  ↓
returns to Twilio
  ↓
Twilio sends WhatsApp response
```

### Common Errors

**"No module named 'utils'"**
- Make sure to run pytest from `backend/` directory
- Or use: `cd backend && pytest ...`

**"Connection timeout"**
- Check Redis URL in config/settings.py
- Check Supabase credentials
- Verify network connectivity

**"Invalid UPI format"**
- UPI must contain @ symbol (e.g., `ravi@upi`)
- Resend correct format

**"Pin code must be 6 digits"**
- Each pin code must be exactly 6 digits
- Format: `560001, 560002, 560003`

### Checking if worker is registered

```python
from utils.db import get_supabase

sb = get_supabase()
result = sb.table("workers").select("id").eq("phone", "+919876543210").execute()

if result.data:
    print("Worker registered ✅")
else:
    print("Worker not found ❌")
```

### Getting worker's preferred language

```python
from utils.db import get_supabase

sb = get_supabase()
worker = sb.table("workers").select("language").eq("phone", "+919876543210").execute()

if worker.data:
    lang = worker.data[0].get("language", "en")
    print(f"Worker's language: {lang}")
```
